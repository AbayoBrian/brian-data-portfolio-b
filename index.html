Government Case Study

✦ Real Organisational Work · Government of Kenya

National Irrigation Authority — Irrigation Information Management System

Baringo County, Kenya · May – Aug 2025 · Python · SQL · Streamlit · Excel

Challenge

The Authority tracked irrigation infrastructure, water usage, and field agent activities across 5 zones using paper forms and disconnected spreadsheets. Monthly reports took 3–5 days to consolidate manually. No validation layer existed — duplicates and missing records were only discovered after submission.

Solution

  • Profiled all existing Excel files to identify data quality issues
  • Cleaned 5,000+ records using Python — standardised fields, resolved duplicates, flagged irrecoverable entries for re-collection
  • Designed a normalised SQLite schema with tables for zones, assets, water usage, and agent records
  • Built automated validation rules: duplicate detection, out-of-range checks, mandatory field enforcement
  • Developed a Streamlit dashboard with zone-level views and exportable monthly templates

Outcome

  • Data accuracy improved by 40% (pre- vs post-implementation audit comparison)
  • Report generation time reduced from 3–5 days to same-day automated export
  • Recurring entry errors dropped by over 60% after validation deployment
  • Dashboard adopted by field supervisors within 3 weeks
  • 3 underperformance incidents investigated using system-generated defect summaries

Kenya Disease Burden & Health Facility Performance

PythonSQL PandasPlotlyLooker Studio
📂 Synthetic · Modelled on WHO Kenya Health Indicators 2018–2025

Analysed 8 years of health facility records across 12 Kenyan regions and 8 diseases to identify high-burden areas and model resource allocation scenarios.

🔍 Key Insight: 2 diseases account for 80% of disease burden. Western region shows 58% higher incidence than national average.
📋 Recommendation: Prioritise malaria prevention budget for high-burden regions. Deploy mobile health units to areas with access scores below 0.5.
  1. Business Problem: Identify high-burden disease regions to guide public health budget allocation
  2. Dataset: Synthetic records for 12 regions × 8 diseases × 8 years, modelled on WHO Kenya data distributions
  3. Data Cleaning: Checked for nulls, validated incidence rates, standardised region names using pandas
  4. SQL Analysis: Wrote queries for Pareto distribution, mortality correlation, and year-on-year trend using SQLite in Python
  5. Visualisation: Built regional map with Plotly; scatter plot for access score vs mortality; trend charts
  6. Dashboard: Exported to Looker Studio via CSV; published interactive HTML maps
  7. Output: Written insight report with 4 prioritised recommendations

Mobile Money Transaction & Credit Risk Segmentation

PythonSQL Scikit-learnRandom ForestPower BI
📂 Synthetic · 50K transactions generated to mirror M-PESA lending patterns

Analysed 50,000+ transactions across 5,000 customers to identify high-risk segments and build a credit risk prediction model (AUC-ROC: 0.85).

🔍 Key Insight: Customers under 25 have 3× higher default rate. Top 20% of customers drive 65% of transaction volume.
📋 Recommendation: Implement lower credit limits for 18–24 age group. Create premium product for medium-value, low-risk segment.
  1. Business Problem: Identify which customers are high default risk before extending mobile loans
  2. Dataset: Synthetic — 5,000 customers, 50K transactions generated using realistic M-PESA behavioural distributions
  3. Feature Engineering: Transaction frequency, recency, average amount, std deviation, age group, income tier
  4. SQL Analysis: Default rates by segment, channel preference, Pareto concentration of transaction volume
  5. Modelling: Random Forest (AUC-ROC: 0.85) vs Logistic Regression baseline (AUC-ROC: 0.71); evaluated with classification report
  6. Segmentation: K-Means (k=4) to identify High-Value/Low-Risk, Medium-Value, Low-Value/High-Risk, and Dormant segments
  7. Output: Risk heatmap by age group and location; written recommendations for credit product design

Clean Cooking Stove Distribution & Field Operations Efficiency

PythonSQL K-MeansPandasPower BI
📂 Synthetic · Modelled on last-mile distribution logistics in Kenya

Analysed 10,000+ delivery records across 16 Kenyan regions to identify logistics bottlenecks and optimise field agent performance using cluster analysis.

🔍 Key Insight: 3 regions cause 80% of delivery delays. Q4 shows 35% higher delivery times due to seasonal road conditions.
📋 Recommendation: Open satellite warehouse in highest-delay region. Implement route optimisation for deliveries exceeding 120km.
  1. Business Problem: Identify which regions and agents cause delivery delays and why
  2. Dataset: Synthetic — 10,000+ delivery records across 16 counties with distance, seasonality, and quantity variables
  3. Data Cleaning: Standardised region names, capped outlier delivery times at 99th percentile, filled missing vehicle type from quantity rules
  4. SQL Analysis: Pareto delay analysis by region; seasonal performance by quarter; stove model delay rates
  5. Root Cause Analysis: Multi-factor regression to weight distance (45%), road condition (31%), and agent workload (24%) as delay drivers
  6. Clustering: K-Means on agent performance (delivery time + rating) to tier agents into 3 performance groups
  7. Output: Satellite warehouse ROI model; route optimisation scenarios; agent retraining recommendations
